1. Introduction

1.1 Purpose

The purpose of this document is to present a description of the web application Fruits On The Web based on the project FrOW.

It will explain the purpose and features of the application, the interface of the software, what it will do and the constraints under which it must operate.

This document is intended for any potential users of the software.

1.2 Intended Audience and Reading Suggestions

This document is intended for any user that might wish to use the Fruits On The Web web application.

1.3 Product Scope

Fruits On The Web has been developed with the intended purpose of being an interactive game for primary school children.

2. Overall Description

2.1 Product Description

The Fruits on the Web web application is based on the FrOW project.

It is a game-type application meant for primary school children who wish to learn about fruits in an interactive way.

Users can play levels with different dificulties in which they must complete the challenges within the given time to obtain the most amount of points.

The developed product can be used on any device with a connection to the Internet and a browser installed.

2.4 Design and Implementation Constraints

Fruits On The Web is developed in node.js and uses HTML, CSS and JavaScript for its visualization engine. It uses mongoDB for data storage.

4.1 Home Page

Users can enter their username, start a new game and load a game, navigate to all other pages, go to the leaderboard page and also change the language to Romanian for the whole app.

4.2 Login Page

Users of the app can login by entering their credentials and therefore access other functionalities of the app or register if they do not have an account and navigate to other pages from this page, or change the password if forgotten.

4.3 Register Page

Guests can create an account and then use it to log into the application and access all the features it provides and navigate to other pages from this page.

4.4 Reset Password Page

Users of the app can reset their password by entering the username. If the username is correct\ they will receive an email on the address provided at registration, containing a token.

4.5 Change Password Page

Users of the app can change their password by entering the token received in the email and typing a new passw. After successful completion, the user is redirected to the login page.

4.6 Help Page

Users can send a message that reports issues with the app using their email and navigate to other pages from this page.

4.7 About Page

Users can read the rules of the game and some data about the authors.

4.8 Select Level Page

Users can select levels based on their difficulty and navigate to other pages from this page.

4.9 Game Page

Users can play the game by matching 3 or more of the same fruit vertically/horizontally, restart the level, quit the level and navigate to other pages from this page. The game is timed and scored. It also features a pause button that stops the timer in case the user wants to take a break from the current level.

4.10 Profile Page

Users are able to see their details and also complete a description about them.

4.11 Leaderboard Page

Users are able to see their scores beginning from the highest score to the lowest. The leaderboard gets updated via RSS constantly so the users only need to refresh the page to see updated information.

4.12 Admin Page

Certain users that have an admin role will be able to see the username, first name, last name, email and score of all the registered users that have used the application. They can also delete any user they'd like and also they can delete all the users at once, emptying the database. This page is also updated constantly and by refreshing the page the lates information gets shown.

5. Use of Technologies: Motivation

5.1 CSS

To make the app as receptible as possible to primary school children, the CSS used includes:

  • Background: The fruity background creates a fun and playful atmosphere, and helps reinforce the theme of the game.
  • Images: The images with fruits creates a interactive and educational environment for the intended users.
  • Colors: The right color scheme helps enhance the overall design, create a fun and engaging experience, and make the game more visually appealing.
  • Fonts: Using a playful and bold font like "Amatic SC" helps create a playful approach for the game.
  • Layout: Aligning the sections into columns makes the page easier to read and navigate.
  • Grid: The grid is used in the game which involves matching fruits 3 by 3.
  • Responsiveness: The app is made responsive so that the intended users(primary school children) can play on different devices.

5.2 Backend: Node.js and Frontend: JavaScript

  • JavaScript is a versatile programming language that runs in the browser and enables dynamic and interactive elements on websites.
  • Node.js, on the other hand, is a runtime environment that allows JavaScript to be executed on the server-side. It provides a powerful platform for building high-performance web applications.
  • JavaScript, combined with Node.js, enables data exchange between the server and the client. Through APIs we can establish communication channels, retrieve data from databases, and send it to the front-end for rendering.

5.2 Backend: Database(mongoDB)

  • MongoDB is a popular NoSQL database that integrates with JavaScript and Node.js for backend development. It provides a flexible and scalable solution for storing and retrieving data in web applications.
  • As a document-oriented database, MongoDB stores data in JSON-like documents, making it a natural fit for JavaScript-based applications.
  • By working with MongoDB's capabilities, JavaScript developers working with Node.js can efficiently store and retrieve data, build scalable backend systems, and deliver robust web applications with a flexible and adaptable data layer.

The score field constantly updates, by summing the scores for each level. The admin field determines if a user is an admin, which results in certain privileges in using the app such as: deleting all the users in the database.
